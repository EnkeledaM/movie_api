myFlix REST API

The myFlix API provides access to information about movies, genres, and directors. It also allows users to register, update their profile, manage a list of favorite movies, and deregister their account.

myFlix API Endpoints

Request URL HTTP Method Response
Get all movies /movies GET Text: list of all movies
Get movie by title /movies/:title GET Text: movie data by title
Get genre by name /genres/:name GET Text: genre description
Get director by name /directors/:name GET Text: director data
Register new user /users POST Text: user registered
Update user /users/:username PUT Text: user updated
Add favorite movie /users/:username/movies/:movieId POST Text: movie added to favorites
Remove favorite movie /users/:username/movies/:movieId DELETE Text: movie removed from favorites
Deregister user /users/:username DELETE Text: user removed

Endpoint Details

GET /movies

  • Description: Returns a list of all movies.
  • Method: GET
  • URL Parameters: None
  • Request Body: None
  • Response Format: Text message (future: JSON array)
  • Example Request: GET http://localhost:8080/movies
  • Example Response: 200 OK – Successful GET request returning data on all movies

GET /movies/:title

  • Description: Returns data about a single movie by title.
  • Method: GET
  • URL Parameters: :title – movie title
  • Request Body: None
  • Response Format: Text message (future: JSON object)
  • Example Request: GET http://localhost:8080/movies/Inception
  • Example Response: 200 OK – Movie data for Inception

GET /genres/:name

  • Description: Returns information about a genre.
  • Method: GET
  • URL Parameters: :name – genre name
  • Request Body: None
  • Response Format: Text message
  • Example Request: GET http://localhost:8080/genres/Thriller
  • Example Response: 200 OK – Genre description

GET /directors/:name

  • Description: Returns information about a director.
  • Method: GET
  • URL Parameters: :name – director name
  • Request Body: None
  • Response Format: Text message
  • Example Request: GET http://localhost:8080/directors/Christopher%20Nolan
  • Example Response: 200 OK – Director information

POST /users

  • Description: Registers a new user.
  • Method: POST
  • Request Body:
{
  "Username": "enkeleda",
  "Password": "test123",
  "Email": "enkeleda@example.com",
  "Birthday": "1973-08-04"
}
  • Example Response: 201 Created – User registered

PUT /users/:username

  • Description: Updates user information.
  • Method: PUT
  • URL Parameters: :username
  • Request Body:
{
  "Username": "enkeleda_new"
}
  • Example Response: 200 OK – User updated

POST /users/:username/movies/:movieId

  • Description: Adds a movie to favorites.
  • Method: POST
  • Example Request: POST http://localhost:8080/users/enkeleda/movies/123
  • Example Response: 200 OK – Movie added to favorites

DELETE /users/:username/movies/:movieId

  • Description: Removes a movie from favorites.
  • Method: DELETE
  • Example Request: DELETE http://localhost:8080/users/enkeleda/movies/123
  • Example Response: 200 OK – Movie removed from favorites

DELETE /users/:username

  • Description: Deregisters a user.
  • Method: DELETE
  • Example Request: DELETE http://localhost:8080/users/enkeleda
  • Example Response: 200 OK – User removed
