myFlix REST API

The myFlix API provides access to information about movies, genres, and directors. It also allows users to register, log in, update their profile, manage favorite movies, and deregister their account. This API uses MongoDB Atlas and Mongoose.

Note: Examples below use http://localhost:8080. Replace example IDs/titles with values from your database.
MovieID format: MongoDB ObjectId (24-hex string), e.g. 65a1f2c3d4e5f67890123456

myFlix API Endpoints

Request URL HTTP Method Request Body Response Authentication
Get all movies /movies GET None 200 OK (JSON array of movies) JWT authentication
Get movie by title /movies/:Title GET None 200 OK (JSON movie object) or 404 JWT authentication
Get genre by name /genres/:Name GET None 200 OK (JSON genre object) or 404 JWT authentication
Get director by name /directors/:Name GET None 200 OK (JSON director object) or 404 JWT authentication
Register new user /users POST JSON (Username, Password, Email, Birthday) 201 Created (JSON user) or 400/422/500 none
Login (get JWT token) /login POST JSON (Username, Password) 200 OK (JSON user + token) or 400 username and password
Update user info /users/:Username PUT JSON (Username, Password, Email, Birthday) 200 OK (JSON updated user) or 404/422/500 JWT authentication
Add favorite movie /users/:Username/movies/:MovieID POST None 200 OK (JSON updated user) or 404/500 JWT authentication
Remove favorite movie /users/:Username/movies/:MovieID DELETE None 200 OK (JSON updated user) or 404/500 JWT authentication
Deregister user /users/:Username DELETE None 200 OK (confirmation message) or 404/500 JWT authentication

Endpoint Details

1) GET /movies

  • Description: Returns a list of all movies.
  • Method: GET
  • Authentication: JWT authentication
  • URL Parameters: None
  • Request Body: None
  • Success Response: 200 OK (JSON array)
  • Business Logic (Mongoose): Movies.find()
GET http://localhost:8080/movies

2) GET /movies/:Title

  • Description: Returns data about a single movie by its title.
  • Method: GET
  • Authentication: JWT authentication
  • URL Parameters: :Title
  • Request Body: None
  • Success Response: 200 OK
  • Error Responses: 404 Not Found
  • Business Logic (Mongoose): Movies.findOne({ Title: req.params.Title })
GET http://localhost:8080/movies/Inception

3) GET /genres/:Name

  • Description: Returns genre info by genre name.
  • Method: GET
  • Authentication: JWT authentication
  • URL Parameters: :Name
  • Request Body: None
  • Success Response: 200 OK
  • Error Responses: 404 Not Found
  • Business Logic (Mongoose): Movies.findOne({ "Genre.Name": req.params.Name })
GET http://localhost:8080/genres/Sci-Fi

4) GET /directors/:Name

  • Description: Returns director info by director name.
  • Method: GET
  • Authentication: JWT authentication
  • URL Parameters: :Name
  • Request Body: None
  • Success Response: 200 OK
  • Error Responses: 404 Not Found
  • Business Logic (Mongoose): Movies.findOne({ "Director.Name": req.params.Name })
GET http://localhost:8080/directors/Christopher%20Nolan

5) POST /users

  • Description: Registers a new user.
  • Method: POST
  • Authentication: none
  • URL Parameters: None
  • Request Body: JSON (Username, Password, Email, optional Birthday)
  • Success Response: 201 Created
  • Error Responses: 400 (user exists), 422 (validation), 500
  • Business Logic (Mongoose): Users.hashPassword() + Users.create()
POST http://localhost:8080/users
{
  "Username": "testuser2",
  "Password": "test1234",
  "Email": "test1@test.com",
  "Birthday": "1990-01-01"
}

Note: Passwords are stored as hashed values in the database (bcrypt), e.g. $2b$10$...

6) POST /login

  • Description: Logs in a user and returns a JWT token.
  • Method: POST
  • Authentication: username and password
  • URL Parameters: None
  • Request Body: JSON (Username, Password)
  • Success Response: 200 OK (JSON with token)
  • Error Responses: 400 Bad Request (invalid credentials)
  • Business Logic: LocalStrategy validates password using user.validatePassword(), then JWT is issued.
POST http://localhost:8080/login
{
  "Username": "testuser2",
  "Password": "test1234"
}

Example Response (200 OK):

{
  "user": {
    "_id": "65b2aa11bb22cc33dd44ee55",
    "Username": "testuser2",
    "Email": "test1@test.com",
    "Birthday": "1990-01-01T00:00:00.000Z",
    "FavoriteMovies": []
  },
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

7) PUT /users/:Username

  • Description: Updates an existing user's information.
  • Method: PUT
  • Authentication: JWT authentication
  • URL Parameters: :Username
  • Request Body: JSON (any fields to update)
  • Success Response: 200 OK
  • Error Responses: 404, 422, 500
  • Business Logic (Mongoose): Users.findOneAndUpdate()
PUT http://localhost:8080/users/testuser2
{
  "Email": "new_email@test.com"
}

Note: If password is updated, the API hashes it before saving.

8) POST /users/:Username/movies/:MovieID

  • Description: Adds a movie to the user's favorites.
  • Method: POST
  • Authentication: JWT authentication
  • URL Parameters: :Username, :MovieID
  • Request Body: None
  • Success Response: 200 OK

9) DELETE /users/:Username/movies/:MovieID

  • Description: Removes a movie from the user's favorites.
  • Method: DELETE
  • Authentication: JWT authentication
  • URL Parameters: :Username, :MovieID
  • Request Body: None
  • Success Response: 200 OK

10) DELETE /users/:Username

  • Description: Deregisters (deletes) a user.
  • Method: DELETE
  • Authentication: JWT authentication
  • URL Parameters: :Username
  • Request Body: None
  • Success Response: 200 OK
